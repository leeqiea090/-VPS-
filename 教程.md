# Debian：Nginx + Certbot 申请证书，并部署 Trojan + Hysteria2（HY2）


> 适用环境：Debian 12/13（含 Trixie）
> 目标：用 **Trojan（443/TCP）** + **Hysteria2（443/UDP）** 共用同一域名与证书
> （免责声明，仅用于技术分享，运用此技术进行任何活动与本人无关）

---

## 0. 前置条件与检查

### 0.1 准备

- 你有一台 VPS（Debian）
- 你有一个域名，并已设置 **A 记录指向 VPS IPv4**
- 确保 VPS 公网可访问 **80/443**
- 我们假设是单个vps，单个域名，且不使用 CDN（如 Cloudflare）

### 0.2 检查域名解析

在 VPS 或本地运行：

```bash
dig +short A 你的域名
dig +short AAAA 你的域名
```

- `A` 必须返回你的 VPS IPv4（例如 `158.247.xxx.xxx`）
- `AAAA` 建议先不要配（避免客户端优先 IPv6 导致排错困难）

## 1 安装 Nginx + Certbot 并签发证书

### 1.1 安装

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

### 1.2 修改 Nginx 默认站点的 `server_name`

编辑默认站点：

```bash
sudo nano /etc/nginx/sites-available/default
```

把 `server_name` 改成你的域名（示例）：

```nginx
server_name 你的域名;
```

检查配置并重启：

```bash
sudo nginx -t && sudo systemctl restart nginx
```

### 1.3 放行端口（HTTP + HTTPS + QUIC）

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 443/udp
sudo ufw status verbose
```

### 1.4 签发证书（HTTP-01）

```bash
sudo certbot --nginx -d 你的域名
```

## 2 让 Nginx 不抢占 443/TCP（留给 Trojan）

Certbot 可能会自动在 Nginx 里添加：

```nginx
listen 443 ssl; # managed by Certbot
```

但 Trojan 需要占用 `443/TCP`，所以 **Nginx 不能占用 443/TCP**。

### 2.1 检查 443/TCP 占用情况

```bash
sudo ss -lntp | grep ':443'
```

如果看到 `nginx` 监听 443，就需要把 Nginx 的 `listen 443 ssl` 注释掉。

### 2.2 注释 Nginx 的 443 ssl（仅保留 80）

编辑：

```bash
sudo nano /etc/nginx/sites-available/default
```

把这些行注释掉（或删除）：

```nginx
# listen 443 ssl; # managed by Certbot
# listen [::]:443 ssl; # managed by Certbot
```

然后 reload：

```bash
sudo nginx -t && sudo systemctl reload nginx
sudo ss -lntp | grep ':443'
```

## 3 部署 Trojan（443/TCP）
## Trojan是一种基于 TLS 的代理协议，通过伪装成正常流量进行转发，具有混淆性和抗封锁能力。我们将使用 Trojan 来监听 443/TCP，提供安全的代理服务。
### 3.1 安装 Trojan（脚本方式）

```bash
sudo apt install -y curl unzip
bash <(curl -fsSL https://raw.githubusercontent.com/trojan-gfw/trojan-quickstart/master/trojan-quickstart.sh)
```

### 3.2 修改 Trojan 配置文件

编辑：

```bash
sudo nano /usr/local/etc/trojan/config.json
```

示例

```json
{
  "run_type": "server",
  "local_addr": "0.0.0.0",
  "local_port": 443,

  "remote_addr": "127.0.0.1",
  "remote_port": 80,

  "password": ["TrojanStrongPassword_ChangeMe"],

  "ssl": {
    "cert": "/etc/letsencrypt/live/你的域名/fullchain.pem",
    "key": "/etc/letsencrypt/live/你的域名/privkey.pem",
    "alpn": ["h2", "http/1.1"]
  }
}
```

### 3.3 启动 Trojan

```bash
sudo systemctl enable --now trojan
sudo systemctl status trojan --no-pager
```

## 4 部署 Hysteria2（HY2，443/UDP）
## Hysteria2 是一个基于 QUIC 协议的代理工具，支持 UDP 传输，具有高性能和低延迟的特点。我们将使用 Hysteria2 来监听 443/UDP，提供快速的代理服务。
### 4.1 安装 Hysteria2

```bash
bash <(curl -fsSL https://get.hy2.sh/)
```

### 4.2 编写 HY2 服务端配置

编辑：

```bash
sudo nano /etc/hysteria/config.yaml
```

示例（字段详见 Hysteria2 官方配置文档）

```yaml
listen: :443

tls:
  cert: /etc/letsencrypt/live/你的域名/fullchain.pem
  key: /etc/letsencrypt/live/你的域名/privkey.pem

auth:
  type: password
  password: Hy2StrongPassword_ChangeMe

obfs:
  type: salamander
  salamander:
    password: Hy2ObfsPassword_ChangeMe
```

### 4.3 启动 HY2

```bash
sudo systemctl enable --now hysteria-server
sudo systemctl status hysteria-server --no-pager
```

### 4.4 确认 HY2 正在监听 443/UDP

```bash
sudo ss -lunp | grep ':443'
```

## 5 Clash / shadowrocket 客户端 YAML 示例（导入用）

> 不同 Clash 内核字段略有差异，下方示例更接近 **mihomo** 的字段命名。

```yaml
mixed-port: 7890
mode: rule
log-level: info

proxies:
  - name: trojan-vultr
    type: trojan
    server: 你的域名
    port: 443
    password: "TrojanStrongPassword_ChangeMe"
    sni: 你的域名
    udp: true
    alpn:
      - h2
      - http/1.1

  - name: hy2-vultr
    type: hysteria2
    server: 你的域名
    port: 443
    password: "Hy2StrongPassword_ChangeMe"
    sni: 你的域名
    obfs: salamander
    obfs-password: "Hy2ObfsPassword_ChangeMe"
    alpn:
      - h3
    skip-cert-verify: false

proxy-groups:
  - name: PROXY
    type: select
    proxies:
      - trojan-vultr
      - hy2-vultr

rules:
  - MATCH,PROXY
```
